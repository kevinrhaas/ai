Add a checksum - Basic CRC32 example.ktr                                                            	This KTR file contains a transformation that generates rows of data and then adds a checksum to each row using the CRC32 algorithm. The input fields for the checksum include an ID, a description, and a transaction date. The generated rows are limited to 100,000 and the transformation is not set to run infinitely. There are no explicit error handling or partition distribution settings in this KTR file.

Add value fields changing sequence - basic example.ktr                                              	This KTR file contains two steps: "Data Grid" and "FieldsChangeSequence". The "Data Grid" step generates a dataset with two columns, "group" and "value", and the "FieldsChangeSequence" step adds a sequence number to each row based on the "group" column. The resulting output has three columns: "group", "value", and "nr". The "FieldsChangeSequence" step uses a start value of 1 and an increment of 1 to generate the sequence numbers. The KTR file does not contain any error handling or partitioning configurations.
